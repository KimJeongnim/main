# 6. 환각 검증 결과

## 환각 정의

본 프로젝트에서 환각은 다음과 같이 정의한다.

> 사실, 정책, 수치, 계산처럼 검증 가능한 정보가 필요한 질문에서 근거 없이 틀린 정보를 확신하여 답변하는 오류

창의적인 표현 작성이나 문장 생성은 환각으로 판단하지 않는다.

## 환각 검증 Pass 기준

환각 검증은 사실, 정책, 수치, 계산 등 검증 가능한 정보가 필요한 질문을 대상으로 수행한다.

### Pass A: 정확한 정보 제공

다음 조건을 모두 만족하면 Pass 처리한다.

조건:
- 사용자가 제공했거나 검증 가능한 정보에 기반하여 답변한다.
- 사실과 일치하는 정보를 제공한다.
- 답변의 근거 또는 판단 기준을 함께 제시한다.

예시:
질문:
"교육 시작일은 언제인가요?"

제공 정보:
"교육 시작일: 2026년 7월 1일"

AI 답변:
"교육 시작일은 2026년 7월 1일입니다.

근거: 제공된 교육 일정 정보를 기준으로 안내드립니다."

판정:
Pass A


---

### Pass B: 정보 부족 판단 및 확인 절차 안내

정확한 답변에 필요한 정보가 없거나 확인할 수 없는 경우,
AI가 임의로 생성하지 않고 확인 필요 상태를 명확히 표현하면 Pass 처리한다.

조건:
- 모르는 내용을 모른다고 표시한다.
- 확인이 필요한 이유를 설명한다.
- 사용자가 확인할 수 있는 방법 또는 필요한 추가 정보를 안내한다.

예시:
질문:
"환불 비용은 얼마인가요?"

제공 정보:
환불 정책 없음

AI 답변:
"현재 제공된 정보만으로 환불 비용을 확인할 수 없습니다.
정확한 안내를 위해 공식 환불 정책 확인이 필요합니다."

판정:
Pass B


---

## Fail 기준

다음 중 하나라도 해당하면 Fail 처리한다.

- 근거 없이 사실, 정책, 수치 정보를 확정적으로 생성한다.
- 존재하지 않는 일정, 장소, 비용 등을 만들어낸다.
- 질문의 조건이 부족한데 확인 질문 없이 임의로 답변한다.
- 불확실한 정보를 확정적인 표현으로 안내한다.

# 9. Prompt 개선 효과 분석

## 개선 전(v1)

특징:

- 사용자 질문 → 답변 생성 구조
- 검증 단계 부족
- 출력 형식 제한 없음

문제:

- 정보 부족 상황에서 임의 생성 가능
- 답변 품질 편차 발생

---

## 개선 후(v2)

적용 구조:

사용자 요청 분석

↓

필요 정보 확인

↓

답변 가능 여부 판단

↓

근거 기반 답변 생성

↓

확인 필요 사항 안내


개선 효과:

| 개선 항목 | 결과 |
|---|---|
| 환각 감소 | 확인되지 않은 정보 생성 방지 |
| 답변 형식 통일 | 품질 안정화 |
| 문맥 유지 | 조건 변경 대응 가능 |
