# 2. 비교 목적

교육기관 수강생 문의 대응 업무는 교육 일정, 장소, 준비물, 출결 기준, 수료 조건 등 사실 기반 정보 안내가 필요한 업무이다.

따라서 단순한 자연어 생성 능력보다 다음 요소가 중요하다.

- 정확한 정보 전달
- 확인되지 않은 정보 생성 방지
- 반복 문의에 대한 일관된 답변
- 운영 정책 변경 대응
- 비용 대비 효율성

본 비교에서는 동일한 업무 입력을 3종 이상의 LLM 모델에 적용하고, 업무 자동화 관점에서 적합한 모델을 선정한다.

# 4. 동일 테스트 프롬프트

모든 모델에 동일한 입력을 사용하였다.

## 테스트 입력

```
당신은 교육기관 수강생 문의 대응 AI입니다.

수강생 질문:
"교육 시작일과 장소를 알려주세요."

현재 제공된 정보:
- 교육 과정명: AI 네이티브 과정
- 교육 기간: 2026년 7월 1일 ~ 2026년 11월 30일
- 교육 장소: 대전정보문화진흥원

답변 형식:
1. 핵심 답변
2. 근거 정보
3. 추가 확인 필요 사항
```

# 8. 최종 선정 모델

## 선정 결과

선정 모델:

```
GPT-5.4 mini(x0.5)
```

---

## 선정 이유

첫째, 교육기관 수강생 문의 대응 업무는 단순 생성보다 정확한 정보 전달과 정책 기반 답변이 중요하다. GPT-5.4 mini(x0.5) 모델은 역할 기반 프롬프트 적용과 출력 형식 제어가 안정적이다.

둘째, 본 프로젝트에서는 환각 방지를 위해 "모르면 확인 필요 표시", "근거 기반 답변", "추가 확인 질문" 구조를 적용한다. GPT-5.4 mini(x0.5) 모델은 이러한 업무 규칙 적용에 적합하다.

셋째, 10턴 이상의 상담 흐름에서 교육 과정명, 사용자 요구사항, 변경 조건을 유지하는 문맥 관리가 중요하다. GPT-5.4 mini(x0.5) 모델은 장기 대화 기반 업무 자동화에 적합하다.

품질 유지를 위해 다음 프로세스를 적용한다.

2. 답변 출력 형식을 제한한다.

3. 교육 정책 데이터는 별도 문서로 관리한다.

4. 중요한 문의는 사람 검토 단계로 전달한다.

5. 복잡한 질문은 여러 단계로 분리한다.

# 10. 정책 변경 대응 전략

## 문제 상황

교육 일정, 환불 정책, 출결 규정 등이 자주 변경될 경우 오래된 정보가 답변에 사용될 가능성이 있다.

---

## 해결 방법

### 데이터 관리 방식 개선

- 정책 정보를 프롬프트 내부에 직접 저장하지 않는다.
- 최신 운영 문서를 기준 데이터로 관리한다.
- 변경 발생 시 데이터만 업데이트한다.

---

### 답변 규칙

AI는 최신 정보 확인이 필요한 경우:

```
현재 제공된 운영 정보 기준으로 안내드립니다.
변경 여부는 운영 담당자 확인이 필요합니다.
```

형태로 안내한다.

---

# 11. 문맥 단절 대응 전략

## 문제 상황

10턴 이상의 상담 과정에서 이전 조건이 누락될 수 있다.

---

## 개선 방법

프롬프트 구조를 다음과 같이 개선한다.

기존 방식:

```
질문 → 답변
```

개선 방식:

```
사용자 요구 확인

↓

현재까지 확정 정보 정리

↓

변경 사항 확인

↓

답변 생성

↓

추가 확인 사항 안내
```

---

## 유지해야 하는 대화 정보

- 교육 과정명
- 교육 일정
- 장소
- 사용자 요청 목적
- 변경된 조건
- 출력 형식 요구사항

# 12. 결론

교육기관 수강생 문의 자동화 업무에서는 생성 능력보다 정확성과 검증 가능한 답변 생성 능력이 중요하다.

비교 결과 GPT-5.4 mini(x0.5) 모델이 업무 요구사항, 프롬프트 제어, 문맥 유지, 환각 방지 측면에서 가장 적합하다고 판단하였다.

다만 실제 운영 환경에서는 최신 정책 데이터 관리와 검증 프로세스를 함께 적용해야 안정적인 자동화 시스템 구축이 가능하다.
